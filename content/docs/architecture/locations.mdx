## Structure

```
Practice
  └── Office (physical building)
        └── Room (space within building)
```

A practice has one or more **offices** (buildings). Each office contains **rooms** (spaces like operatories, sterilization rooms, etc.).

## Offices

The `offices` table tracks physical locations:

- `is_primary` — marks the main office
- Full address fields for multi-location practices
- Practice-scoped via `practice_id`

## Rooms

Rooms are spaces within an office. They have two name fields:

- `room_type` — stable identifier used for icon mapping and default seeding (enum: `lobby`, `operatory`, `sterilization_room`, `xray_room`, etc.)
- `name` — display label, renameable by practice admins

### Default room types

| Type | Default name |
|------|-------------|
| `lobby` | Lobby |
| `front_office` | Front Office |
| `waiting_area` | Waiting Area |
| `operatory` | Operatory |
| `sterilization_room` | Sterilization Room |
| `xray_room` | X-Ray Room |
| `lab` | Lab |
| `consultation_room` | Consultation Room |
| `supply_storage` | Supply / Storage |
| `break_room` | Break Room |
| `restroom` | Restroom |
| `other` | (custom) |

### Custom rooms

Practice admins can add rooms beyond the defaults. Custom rooms have `is_custom = true` and `room_type = 'other'`.

## Task context

Tasks can optionally reference a location:

- `tasks.room_id` → specific room
- Equipment is also room-scoped via `equipment.room_id`
