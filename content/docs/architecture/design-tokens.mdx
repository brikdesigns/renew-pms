Renew PMS uses the Brik Design System (BDS) token system for all color, typography, and spacing values. Colors are never hardcoded — every value references a CSS custom property defined in `figma-tokens.css`.

This page documents the **data-driven color tokens** used throughout the application.

## Department colors

Each department has a color key stored in `departments.color` (e.g. `"blue"`, `"green"`). This key maps to a set of three CSS custom properties per color:

| Token | Purpose | Example usage |
|-------|---------|---------------|
| `--color-department-{key}` | Base/saturated | Table dot indicators, charts |
| `--color-department-{key}-light` | Light/pastel | Card backgrounds, avatar fills, department tags |
| `--color-department-{key}-text` | Contrast text | Text on light backgrounds, avatar initials |

### Color derivation chain

Department color flows through the data model to style UI elements automatically:

```
departments.color ("blue")
    │
    ├── Department cards → colored dot (light token)
    │
    ├── practice_role_types.department_id (FK)
    │       │
    │       ├── Role cards → avatar background (light token + text token)
    │       │
    │       └── practice_members.practice_role_id (FK)
    │               │
    │               ├── User cards → avatar background (light token + text token)
    │               └── User cards → department tag (light token + text token)
    │
    └── Tables → department color dot indicator (base token)
```

This means changing a department's color in settings automatically updates every role card, user card, and avatar that belongs to that department. No manual sync required.

### Fallback behavior

- **Role with no department** (`department_id` is null): avatar falls back to `--border-muted` (neutral gray)
- **Unknown color key**: defaults to `blue` token set
- **User with no role** (`practice_role_id` is null): no department tag shown, avatar uses neutral gray

|---------|-----------------|----------|
| `positive` | Success / active | Active accounts, completed tasks, active templates |
