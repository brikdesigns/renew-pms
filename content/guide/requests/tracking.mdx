## Request status

Every request moves through a lifecycle:

| Status | Meaning |
|--------|---------|
| **Submitted** | Newly created, waiting for admin review |
| **In Review** | An admin is evaluating the issue |
| **In Progress** | Work has started on the fix |
| **Waiting on Vendor** | An external vendor has been contacted (for high/critical issues) |
| **Resolved** | The issue has been fixed |
| **Closed** | Finalized and archived |

## The Requests board

Requests are displayed in a kanban-style board with columns for each status. You can see all requests submitted by anyone at your practice.

## Notifications

You'll receive notifications when:
- **Your request's status changes** — e.g., moves from Submitted to In Review
- **You're assigned to a request** — an admin has asked you to handle it

Notifications appear as a badge on the bell icon in the top bar. Click the bell to see your recent notifications, or click **Mark all read** to clear them.

## Viewing request details

Click any request card to see its full details, including:
- Who submitted it and when
- Current status and urgency
- Location and equipment info
- Assigned team member or vendor (if any)
- Resolution notes (once resolved)
